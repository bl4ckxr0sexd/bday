# 🎉 Birthday Invitation Workflow

A reusable, three-part invitation flow.

```
Email (email.html) ──▶ Landing page (index.html) ──▶ Google Drive download
                              │
                              └──▶ tracking event ──▶ Apps Script (webapp.js) ──▶ Google Sheet
```

Two things you configure, in two different ways:
- **Landing page** — edit `.env`, then the build injects it into `public/index.html`.
- **Email letter** — a standalone file; edit the values at the top of `email.html` directly.
  (Email clients can't run a build, so the email carries its own config.)

## Project structure

```
birthday/
├── email.html            ← STANDALONE email — edit values at the top, then paste into your mail tool
├── .env                  ← config for the LANDING PAGE (name, links, party details)
├── .env.example          ← documented reference copy
├── build.js              ← injects .env into the landing template (no dependencies)
├── package.json          ← `npm run build`
├── vercel.json           ← Vercel build config
├── webapp.js             ← Google Apps Script (the tracking receiver)
├── templates/
│   └── index.template.html   ← landing page source
└── public/               ← GENERATED by the build (index.html)
```

You never edit `public/` — it's regenerated from `templates/` + `.env`.

## Configurable values

### Landing page (`.env`)

The landing page is intentionally short: one screen showing the invitation and
the **Download Invitation** button — all the party details live in the
downloadable file, so there's nothing to scroll past.

| Key                 | What it controls                                            |
|---------------------|------------------------------------------------------------|
| `COUSIN_NAME`       | Birthday person's name (landing page)                      |
| `DOWNLOAD_URL`      | Google Drive link for the downloadable invitation          |
| `TRACKING_ENDPOINT` | Apps Script Web App URL (`/exec`). Blank = tracking off     |
| `PARTY_DATE` / `PARTY_TIME` | Used only for the link-preview text when shared    |

### Email letter (`email.html`)
`email.html` is a **plain, static, send-ready HTML letter** — no scripts, no
build. Edit the values by hand (each spot is marked with an `EDIT:` comment; the
list is at the top of the file):

| Marker          | What to change                          | Times it appears |
|-----------------|-----------------------------------------|------------------|
| `EDIT: NAME`    | birthday person's name                  | 4                |
| `EDIT: GREETING`| greeting ("Hi there!") — or a name      | 1                |
| `EDIT: DATE`    | the date line                           | 1                |
| `EDIT: TIME`    | the time line                           | 1                |
| `EDIT: URL`     | landing-page link (in 3 `href` spots)   | 3                |

**How to send it:** feed the raw contents of `email.html` to your SMTP sender as
the HTML body, with `Content-Type: text/html; charset=UTF-8`. Because it's
static HTML with inline styles, it renders correctly in email clients as-is.

## Setup — do these in order

### 1. Deploy the data receiver (`webapp.js`)
Follow the comment block at the top of `webapp.js`. You'll get a **Web App URL**
ending in `/exec`. Put it in `.env` as `TRACKING_ENDPOINT`.

### 2. Fill in `.env`
Set `COUSIN_NAME`, `DOWNLOAD_URL`, and the party details.

### 3. Deploy the landing page to Vercel
`vercel.json` already tells Vercel to run the build and serve `public/`.

**CLI**
```bash
npm run build        # optional: preview locally by opening public/index.html
vercel               # deploy a preview
vercel --prod        # promote to production
```

**Dashboard:** push this folder to a Git repo and "Import Project" on
[vercel.com](https://vercel.com), or drag-and-drop the folder. Vercel runs
`node build.js` and serves `public/` automatically.

Copy your live URL — you'll use it in the email in step 4.

### 4. Send the email
Edit the `EDIT:` spots in `email.html` (name, greeting, date, time, and your
Vercel URL from step 3), then send the file's raw HTML as the message body
through your SMTP sender (`Content-Type: text/html; charset=UTF-8`).

## Local development
```bash
npm run build         # regenerate public/index.html from the template + .env
open public/index.html
```
For the email, just open `email.html` in a browser to preview.

## Notes
- **Zero dependencies** — the build is plain Node, so Vercel builds fast.
- **Tracking** collects coarse network info (IP / ISP / country) for guest
  analytics only. If `ipapi.co` is rate-limited the page still loads and still
  sends events (with `unknown` network fields).
- `.env` is safe to commit for this template (no real secrets); `public/` and
  `node_modules/` are gitignored.
